Computational Thinking with Python — Part II

Class 1 — Programs as Systems
Start with a real system, then see how simple Python structures combine to model more complicated problems.

Class goal

Most programming courses begin with syntax. Today we begin with a different idea: a program can be understood as a system. A system has a current condition, some ways its condition changes, and some truths that should remain true as time passes.

We will use three connected questions all semester:

STI ideaCore question
StateWhat information represents the current condition of the system?
TransitionsHow does the state change?
InvariantsWhat rules must remain true as the system changes?

Big idea: even before we look at code, we can already talk about a system in terms of its state, its transitions, and its invariants.

The STI model

A computational system = current state + transitions + invariants that must hold

Programming becomes easier when we stop seeing code as just a list of statements and instead ask: what system are we trying to represent?

Insight

Build-up sequence: one small idea at a time

The examples below grow by adding just one new idea at a time. Notice how each new structure expands what kind of system we can represent.

Step 1 — A variable stores one piece of state

score = 0

Step 2 — An expression computes a new value

score = score + 10

Step 3 — A conditional lets a change happen only sometimes

if answer == "correct": score = score + 10

Step 4 — A loop lets the system repeat a transition

for answer in answers: if answer == "correct": score = score + 10

Step 5 — A dictionary lets us store structured state

student = {"name": "Ana", "score": 30, "section": "A"}

Step 6 — A function packages a reusable change

def add_points(score, points): return score + points

Exercise 1.6 — What new capability is added at each step?

For each step above, write one sentence describing what the new structure allows us to do that earlier steps could not do as easily.

Exercise 1.7 — Which steps are most connected to state, which to transitions, and which to invariants?

Key message

Loops, variables, conditionals, collections, and functions are not isolated tricks. They are the pieces from which system behavior is built.

Design habit

When a system feels “complicated,” try breaking it into small structures you already understand.
