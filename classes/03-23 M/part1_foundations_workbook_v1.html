2. Computers Store Data and Change Data

Nearly every program can be understood using two questions:

  • What information is stored?
  • What operations change that information?

A spreadsheet stores numbers and updates them when formulas run. A game stores positions and scores and updates them when the player acts. A banking program stores balances and changes them when transactions occur. A portfolio program stores clients, positions, and cash, then changes them when a trade is recorded.

balance = 100
print(balance)
balance = balance + 25
print(balance)

Before running the code, predict the output. Then run it. Then change the starting value and rerun.

Questions worth asking

  • What value is stored before the update?
  • What value is stored after the update?
  • Which line changes the stored value?
  • What value is produced by balance + 25?
  • Would the same pattern hold if we updated the value twice?

Section 2 Questions

  1. In the example above, which line performs the update?
  2. What is the difference between the value stored before the update and the value produced by balance + 25?

⚠ Warning Panel

When you read code, do not treat all lines as doing the same thing. Ask:

  • Is this line computing a value?
  • Is this line creating or rebinding a reference?
  • Is this line changing an existing object?
  • Is this line controlling the flow of execution?

4. Working with Strings

A string is a sequence of characters. Students need a basic toolkit for manipulating strings because strings appear everywhere: names, menu choices, file paths, stock tickers, dates, and reports.

text = "Babson College"

Indexing and slicing

The general slicing pattern is:

text[start:stop:step]

start is where the slice begins, stop is where it ends (not included), and step is how many positions to move each time.

print(text[0])      # first character
print(text[2:8])    # from index 2 up to but not including 8
print(text[2:])     # from index 2 to the end
print(text[:5])     # from the beginning up to but not including 5
print(text[::2])    # every second character
print(text[-1])     # last character
print(text[-3:])    # last 3 characters

Negative values count backward from the end of the string. So -1 means “last position,” -2 means “second-to-last,” and so on.

text = "Babson" index: 0 1 2 3 4 5 char : B a b s o n neg idx: -6 -5 -4 -3 -2 -1

join matters because many programs build text from pieces.

print(sentence)

f-Strings

This workbook teaches a small, practical subset of the part that comes after the colon :.

f"{value:[fill][align][width][.precision][type]}"

The 4 slots (in order)

  1. Fill + Align (optional)
  2. Width (optional; minimum field width)
  3. .Precision (optional)
  4. Type (optional)

Align

  • < left
  • > right
  • ^ center

Fill

Any single character, but it only works when align is present. Examples: 0>, .^, -<.

Width

A number like 10 meaning minimum width.

Precision

.2 is common with floats, so .2f means 2 digits after the decimal.

Type (common)

  • d integer
  • f fixed-point float
  • % percent

Golden examples

n = 42
x = 12.5
p = 0.3789
name = "Alice"

print(f"{n:>5}")       # right-align, width 5
print(f"{n:0>5}")      # zero-fill, right-align, width 5
print(f"{x:.2f}")      # 2 decimals
print(f"{x:8.2f}")     # width 8, 2 decimals
print(f"{p:.1%}")      # percent, 1 decimal
print(f"{name:^10}")   # center, width 10

Using f-strings to build a small table

ticker = "AAPL"
shares = 10
price = 205.4

print(f"{'Ticker':^10} {'Shares':^10} {'Price':^10}")
print(f"{ticker:^10} {shares:^10d} {price:>10.2f}")

6. Working with Tuples

A tuple is an ordered grouping that is typically not changed.

point = (3, 5)
date = (2026, 1, 20)
location = (42.3, -71.2)

Tuples support indexing and iteration, but not list-style mutation methods like append().

Tuples are useful for communication and defensive programming. They tell the reader that the values represent one fixed concept.

8. Variables, References, and Objects

A variable is not an object. A variable is a label that refers to an object. It is better to think of a variable as a label than as a box.

variable name → reference → object
a = [1, 2]
b = a

print(id(a))
print(id(b))

This is aliasing. Two variables are different names for the same object.

9. Mutability, Immutability, Aliasing, and Copying

This is one of the most important conceptual sections in the workbook.

Some objects can change after they are created. Lists and dictionaries are common mutable objects.

Integers, floats, strings, and tuples behave as immutable objects in our examples.

When the value of an immutable object appears to change, Python creates a new object and the variable starts referring to that new object.

a = [1, 2]
b = a
b.append(3)
print(a)
print(b)
x = 10
y = x
x = 20
print(x)
print(y)
a = [1, 2, 3]
b = a.copy()
b.append(4)
print(a)
print(b)
ActionWhat happens
MutationThe existing object changes
RebindingThe variable starts referring to a different object
AliasingTwo variables refer to the same object
CopyingA new object is created with similar contents

10. Expressions, Statements, and Operations

These distinctions matter because they help you debug and reason more precisely about what code is doing.

CategoryMeaningExamples

Here, len(items) + 3 is the expression. The function call and the + are operations inside that expression. The assignment is the statement.

Be careful not to confuse not x with x != y.

not is_member      # negate a truth value
x != y             # compare two values for inequality

Arithmetic operators

+   -   *   /   //   %   **

Evaluation is usually left to right, but ** behaves differently. When precedence matters, use parentheses.

3 + 4 * 5
(3 + 4) * 5
2 ** 3 ** 2

Selection with if / elif / else

score = 75

if score >= 90:
    print("A")
elif score >= 80:
    print("B")
elif score >= 70:
    print("C")
else:
    print("Below C")

Python checks conditions from top to bottom and stops once one branch succeeds.

De Morgan's Laws

These are two very useful rewrite rules for boolean expressions:

not (A and B) == (not A) or (not B)
not (A or B)  == (not A) and (not B)

Truth tables are an important tool for checking whether two logical expressions are equivalent.

15. Functions, Methods, and Function Signatures

A function packages a computation behind a name. The goal is not just shorter code. The goal is clearer thinking, less repetition, and better decomposition.

Why functions matter: a report writer example

# repeated code
ticker = "AAPL"
shares = 10
price = 205.4
value = shares * price
print(f"{ticker:^10} {shares:^10d} {price:>10.2f} {value:>12.2f}")

ticker = "MSFT"
shares = 5
price = 417.2
value = shares * price
print(f"{ticker:^10} {shares:^10d} {price:>10.2f} {value:>12.2f}")
def print_header():
    print(f"{'Ticker':^10} {'Shares':^10} {'Price':^10} {'Value':^12}")
    print("-" * 46)

def print_row(ticker, shares, price):
    value = shares * price
    print(f"{ticker:^10} {shares:^10d} {price:>10.2f} {value:>12.2f}")

def print_report(positions):
    print_header()

    for ticker, shares, price in positions:
        print_row(ticker, shares, price)

positions = [
    ("AAPL", 10, 205.4),
    ("MSFT", 5, 417.2),
    ("GOOG", 3, 152.8)
]

print_report(positions)
def add(a, b):
    return a + b

print(add(3, 4))
  • Function signature: add(a, b)
  • Parameters: names in the definition
  • Arguments: values passed in the call
  • Return value: the value sent back to the caller
def square(x):
    return x * x

def show_square(x):
    print(x * x)
len(items)       # function call
items.append(3)  # method call

17. Files and JSON

Programs often need to save data between runs. Files let information persist after the program stops.

Reading text files

with open("data.txt", "r") as f:
    print(f.read())

with open("data.txt", "r") as f:
    print(f.readline())
    print(f.readline())

with open("data.txt", "r") as f:
    lines = f.readlines()
    print(lines)

Writing and appending

When writing multiple lines, remember to add where needed.

with open("notes.txt", "w") as f:
    f.write("Hello\\n")
    f.write("World\\n")

with open("notes.txt", "a") as f:
    f.write("Another line\\n")

File position and seek()

with open("data.txt", "r") as f:
    print(f.read(5))
    f.seek(0)
    print(f.read(5))

In beginner text-file work, seek() is not common. It is more commonly discussed with binary files.

What is a binary file?

A text file stores characters. A binary file stores raw bytes. Images, audio files, executable files, and many custom file formats are binary files.

First pattern: CSV

a) Create the object in Python

positions = [
    ("AAPL", 10),
    ("MSFT", 5)
]

print(positions)

b) Write it out to a CSV file

positions = [
    ("AAPL", 10),
    ("MSFT", 5)
]

with open("positions.csv", "w") as f:
    f.write("ticker,shares\\n")
    for ticker, shares in positions:
        f.write(f"{ticker},{shares}\\n")

c) Read it in from CSV

with open("positions.csv", "r") as f:
    print(f.read())

d) Recreate the object

positions = []

with open("positions.csv", "r") as f:
    next(f)  # skip header

    for line in f:
        ticker, shares = line.strip().split(",")
        positions.append((ticker, int(shares)))

print(positions)

Second pattern: JSON

a) Create the object in Python

positions = [
    {"ticker": "AAPL", "shares": 10},
    {"ticker": "MSFT", "shares": 5}
]

print(positions)

b) Write it out to a JSON file

import json

positions = [
    {"ticker": "AAPL", "shares": 10},
    {"ticker": "MSFT", "shares": 5}
]

with open("positions.json", "w") as f:
    json.dump(positions, f, indent=2)

c) Read it in from JSON

with open("positions.json", "r") as f:
    print(f.read())

d) Recreate the object

import json

with open("positions.json", "r") as f:
    positions = json.load(f)

print(positions)

With CSV-like text, you often have to split strings, convert types, and build the object manually. With JSON, the structure is already represented. That is one reason JSON makes life easier.

19. Python Mental Model Summary

This section pulls together the main ideas from Part I into one place. Think of it as a compact reference page you can keep open while coding.

Variable → Reference → Object

A variable is not an object. A variable is a label that refers to an object.

x = 10 y = x x = 20 After rebinding: x ──► 20 y ──► 10

Mutation vs Rebinding

mutation → object changes rebinding → variable refers to a new object
numbers.append(4)        # mutation
numbers = numbers + [4]  # rebinding

Objects contain data and operations

list data → [1, 2, 3] operations → append(), pop(), sort() string data → "Babson" operations → upper(), split()

Three common interfaces in Python

object[key] object.method() function(object)
portfolio["AAPL"]   # key lookup
numbers.append(4)   # method call
len(numbers)        # function call

Expressions vs Statements vs Operations

Expression → produces a value Statement → performs an action Operation → transforms data

Core control-flow tools

if / elif / else → decisions for → iterate over elements range() → generate sequences break → exit loop continue → skip iteration

Data structures you now know

StructureExampleWhen used
list[1, 2, 3]ordered sequence
tuple(x, y)fixed grouped data
dictionary{"AAPL": 10}key → value lookup
string"text"text data

Programs manage state

state → stored data transitions → code that changes state invariants → conditions that must remain true

Transition to Part II: State–Transition–Invariant (STI)

Programs often model real systems. To model a system well, we need a better vocabulary for describing how the system works.

IdeaMeaningBank account example
StateInformation describing the systembalance
TransitionsOperations that change the statedeposit, withdraw
InvariantsProperties that must remain truebalance should not become negative
Example: Tic-Tac-Toe State: board current player Transitions: place mark switch player Invariants: each square is X, O, or blank only one mark per square

Part I was about understanding how Python behaves. Part II will begin with abstraction, decomposition, pattern recognition, and algorithms, then gradually formalize system descriptions using State, Transitions, and Invariants.

Closing thought: By the end of this part, you should not expect to know all of Python. That is not the goal. The goal is to understand enough syntax, structure, and behavior to reason about programs and prepare for computational modeling.
