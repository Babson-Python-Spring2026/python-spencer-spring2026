Student handout — Analyze Runs

Main idea: In this assignment, you must use AI, but you must use it well. You are not being graded only on the final code. You are also being graded on whether you stayed in control of the thinking.

The Problem

Write a function:

analyze_runs(nums)

where nums is a list of integers.

Your function should return a dictionary with this structure:

{
    "longest_increasing_run": ?,   # length of longest strictly increasing contiguous run
    "longest_decreasing_run": ?,   # length of longest strictly decreasing contiguous run
    "num_increasing_runs": ?,      # number of increasing runs
    "num_decreasing_runs": ?,      # number of decreasing runs
    "longest_run_values": ?        # the actual values of the longest run (either type)
}

Definition

A run is a contiguous sequence of values.

For example, in the list below:

[3, 5, 7, 2, 1, 4]

the increasing runs are [3, 5, 7] and [1, 4], and the decreasing run is [7, 2, 1].

Example

nums = [3, 5, 7, 2, 1, 4]

One valid return value would be:

{
    "longest_increasing_run": 3,
    "longest_decreasing_run": 3,
    "num_increasing_runs": 2,
    "num_decreasing_runs": 1,
    "longest_run_values": [3, 5, 7]
}

If there is a tie for the longest run, you may return either tied run in longest_run_values.

Part 1: Solve the Problem Using STI First

Before you ask AI for code, you must first think about the problem using the State–Transition–Invariant (STI) framework.

State

What variables must be tracked while scanning through the list?

For example, you may need to track the current run, its direction, its length, and the best answers seen so far.

Transitions

What happens when the next number causes the run to continue, end, or switch direction?

Think carefully about what should happen when two adjacent values are equal.

Invariants

State at least 4 invariants that should remain true throughout your algorithm.

Examples of the kind of thing you should think about:

  • The current run length is always at least 1 once a run has started.
  • The longest run found so far is never shorter than the current best answer already recorded.
  • Run counts should only increase when a run has been clearly identified.
  • The values stored in longest_run_values should match the recorded longest run length.

Part 2: How to Use AI Correctly

You are required to use AI for this homework. But you should use AI as a thinking partner, not as a machine that simply writes code for you.

What good AI use looks like

  • Start with the problem, not the code.
  • Tell the AI what you think the state variables might be.
  • Ask the AI what assumptions it is making.
  • Ask the AI what edge cases you may be missing.
  • Refine the model before you ask for implementation.
  • If the AI gives code, ask it to explain the code line by line.

What poor AI use looks like

  • Pasting the problem and saying, “solve this.”
  • Submitting code you do not understand.
  • Skipping STI and jumping directly to implementation.
  • Letting the AI make hidden assumptions for you.

Key takeaway: Strong AI use means you drive the design. The AI should help you test ideas, expose ambiguity, and explain reasoning. It should not be the one in control.

How to Label Your Chat Transcript

At the start of your AI session, type this instruction:

For the rest of this conversation, prefix every one of my messages with "USER:"
and every one of your responses with "AGENT:".

This usually works well, but it is not perfect. The AI may occasionally forget. Because of that, when you copy and paste your transcript for submission, you must check that the labels were added correctly.

Before submitting your transcript, check the following:

  • Each of your prompts begins with USER:
  • Each AI response begins with AGENT:
  • The conversation is in the correct order
  • No large sections are missing
  • The transcript clearly shows your thinking and your interaction with the AI

Part 3: Implementation Requirements

Your Python solution must:

  • Use a loop
  • Track state explicitly
  • Correctly detect increasing and decreasing runs
  • Handle edge cases carefully, especially equal adjacent values
  • Return the dictionary in the required format
