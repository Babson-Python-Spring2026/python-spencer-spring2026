Main Goal
Today we begin object-oriented programming. The goal is to see OOP as a continuation of the systems we have already been building with dictionaries, lists, and functions.

1. Opening Motivation

Up to this point, we have built systems using data structures and functions. That works. But as systems get larger, we often want a cleaner way to keep state and behavior together.

Class Discussion
  1. In our earlier portfolio work, where did the state live?
  2. Where did the logic that updated the state live?
  3. What became awkward once we had many functions and many data structures?
  4. What kinds of mistakes were easy to make?
  5. What would it mean for the system itself to own the data it needs?
Motivating Example
position = {
    "ticker": "AAPL",
    "shares": 100,
    "avg_cost": 150.00
}

def buy(position, shares, price):
    position["avg_cost"] = (
        position["shares"] * position["avg_cost"] + shares * price
    ) / (position["shares"] + shares)
    position["shares"] += shares
Think About It
  • What is the state here?
  • What is the transition?
  • What invariant might we want to protect?
  • What if a different function updates shares incorrectly?

2. Core Vocabulary

Class

A class is a blueprint for creating objects.

Object

An object is a particular instance of a class.

Attributes

Attributes are values stored inside an object. They represent the object’s state.

Methods

Methods are functions attached to a class that operate on an object’s state.

__init__

__init__ runs when an object is created and sets up the object’s initial state.

self

Every instance method takes self first. self means the object the method is working on.

4. Minimal Rules

Rule Meaning
Every instance method begins with self The method is working on a particular object.
Attributes are usually written as self.attribute_name The data is stored on the object.
__init__ sets the object up This is where we establish initial state.
A method call uses dot notation dog1.birthday() asks that object to perform that action.

7. Guided Build 3 — Counter

class Counter:
    def __init__(self, start):
        self.count = start

    def increment(self):
        self.count += 1

    def decrement(self):
        self.count -= 1

    def reset(self):
        self.count = 0

Questions
  • Why is this a better example than just using a plain integer?
  • What makes this object stateful?
  • What transitions are available?
  • What invariant might we add if we wanted the counter never to go below zero?
Interesting Observation
Integers are immutable. When we do: 
self.count += 1
the old integer is not being changed in place. Instead, a new value is created and the object’s attribute is updated to refer to that new value.

10. Bridge Back to the Portfolio System

class PortfolioSystem:
    def __init__(self, prices_dates, prices_tickers, ticker_universe, mkt_dates):
        self.prices_dates = prices_dates
        self.prices_tickers = prices_tickers
        self.ticker_universe = ticker_universe
        self.mkt_dates = mkt_dates

    def get_num_market_dates(self):
        return len(self.mkt_dates)

    def get_num_tickers(self):
        return len(self.ticker_universe)

    def is_valid_ticker(self, ticker):
        return ticker in self.ticker_universe
