Lab 04 - Reverse Engineer a Game with AI

Computational Thinking with Python

Overview

In this lab you will be given a compiled Python game file in labs/lab_04. Your job is to reverse engineer the game and recreate it as a readable Python source file.

You may use AI as part of the process. In fact, that is part of the point of the lab. But AI is not being used here as a shortcut around thinking. It is being used as a tool to help you analyze a system, describe its behavior, and reconstruct that behavior in code.

Main goal: Use computational thinking and AI together to infer how the game works, then produce your own Python implementation of that same game.

What you are practicing

  • Program comprehension
  • System decomposition
  • Reasoning about state, transitions, and invariants
  • Using AI as a structured problem-solving tool
  • Translating observed behavior into Python code

Your task

You will start with the compiled game file provided in labs/lab_04.

You must determine:

  • What kind of game it is
  • What the user sees and inputs
  • What data the program must be storing
  • What major operations or functions the game must perform
  • What rules must remain true throughout execution

Then you will write a Python source file that recreates the game.

Required deliverables

Add the following two items to labs/lab_04:

  1. Your Python source file for the game
    This should be your reconstructed .py implementation.
  2. A copy of your AI conversation or conversations
    This should show how you used AI to help analyze and reconstruct the program.
Both files are required. Submitting only the Python file is not enough.

Important constraint

The goal is not to ask AI to generate a generic version of the game from scratch. The goal is to use AI to help you infer and rebuild this specific program.

In other words, the intellectual task is reverse engineering, not just code generation.

How to approach the lab

Step 1: Observe the program

  • Run the compiled game file.
  • Try different inputs.
  • Pay attention to prompts, outputs, symbols, win conditions, and loss conditions.
  • Take notes on what you think the program is doing.

Step 2: Infer the system

Before writing code, describe the program in terms of STI:

  • State: What information must the program store?
  • Transitions: What actions cause the state to change?
  • Invariants: What must remain true while the program runs?

Step 3: Use AI well

Use AI to help you think clearly. For example, you might ask AI to help you:

  • Identify the likely state of the program
  • List the functions the game probably needs
  • Suggest a decomposition into smaller parts
  • Check whether your reasoning makes sense
  • Help write or revise code once you understand what the code should do

Step 4: Build your version

  • Write a readable Python implementation.
  • Use meaningful variable and function names.
  • Test your code carefully.
  • Revise until the behavior matches the original game as closely as possible.

What your AI transcript should show

Your transcript should make it clear that you were using AI as a problem-solving partner, not as a black box.

It should show evidence of things like:

  • Describing observed behavior
  • Asking AI to help infer state and rules
  • Breaking the game into parts
  • Revising prompts when the first answer is vague or wrong
  • Checking whether generated code actually matches the target program

What to avoid

  • Do not submit a random or generic version of the game.
  • Do not submit code you do not understand.
  • Do not remove the AI transcript requirement.
  • Do not assume that because AI produced code, that code is correct.
You are responsible for the final code you submit, whether AI helped produce it or not.

How your work will be evaluated

Category What I am looking for
Behavioral match Your program behaves like the original game in its main features and rules.
Program understanding Your reconstruction reflects understanding of what the system stores and how it changes.
Code quality Your Python code is readable, organized, and reasonably well structured.
Use of AI Your transcript shows thoughtful use of AI as part of analysis and reconstruction.
Completeness You submitted both required deliverables in the correct folder.

A good way to think about this lab

Imagine that someone handed you a working program but not the source code. Could you figure out what system it implements?

That is what good programmers do all the time. They study behavior, infer structure, identify assumptions, and reconstruct logic.

In this lab, AI can help you. But the real work is still yours: observing carefully, reasoning clearly, and rebuilding the system.
