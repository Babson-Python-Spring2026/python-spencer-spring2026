1. Program Description

The shopping cart program allows a user to load an existing cart or create a new cart. The user can then buy items from the Babson store inventory, view the current cart, checkout, or exit. Shopping carts are stored in a JSON file so that the cart can be reloaded later.

General Program Flow

Start program ↓ Load inventory from inventory.json ↓ Load saved carts from shopping_carts.json ↓ User loads existing cart or creates new cart ↓ Repeatedly display cart and menu ↓ User buys item, checks out, or exits ↓ Save carts back to shopping_carts.json

Classes

Class Purpose
ShoppingCart Represents one shopping cart. It has a cart name and a list of Item objects.
Item Represents one item in a cart. It has quantity, description, and price.

Main Functions and Methods

Function / Method General Description
ShoppingCart.displayCart() Displays the current cart contents.
ShoppingCart.getItemNo() Gets and validates the item number selected by the user.
ShoppingCart.getQuantity() Gets and validates the quantity selected by the user.
ShoppingCart.buyItem(inventory) Lets the user buy an item and updates the cart.
ShoppingCart.checkOut() Computes the total amount owed and clears the cart.
Item.__add__(other) Combines two compatible items into one item with a combined quantity.
loadInventory() Loads inventory from JSON and converts inventory keys from strings to integers.
loadShoppingCarts() Loads saved carts from JSON and reconstructs ShoppingCart and Item objects.
storeShoppingCarts(shoppingcarts) Converts cart objects back into JSON-compatible dictionaries and writes them to disk.
cartName(shoppingcarts) Allows the user to load an existing cart or create a new one.
shoppingMenu() Main controller function for the shopping cart program.

2. Files and Data

inventory.json

The inventory file contains the store inventory. In the JSON file, the inventory keys are strings such as "1001", "1002", and so on. After loading the file, the program converts those keys to integers. 

{
  "1001": {
    "inventory_quantity": 25,
    "description": "Hooded Sweatshirt",
    "price": 58.00
  },
  "1002": {
    "inventory_quantity": 25,
    "description": "Long Sleeve T-Shirt",
    "price": 28.00
  }
}
Representation issue: JSON object keys are strings. The program converts those keys to integers at runtime because integer keys make the item-number logic clearer.

shopping_carts.json

The initial shopping carts file may be empty: 

{}

After carts are created and items are purchased, the file stores a dictionary where each key is a cart name and each value is a list of item dictionaries. 

{
  "bob": [
    {
      "quantity": 2,
      "description": "Hooded Sweatshirt",
      "price": 58.00
    }
  ]
}

TODO 1 — Complete Item.__add__() OOP method

Add an __add__ method to the Item class. It should combine two compatible item objects.

Two items are compatible if they have the same description and the same price. If they are compatible, return a new Item object with the combined quantity. If they are not compatible, raise a ValueError. 

# Example:
# Item(2, "Mug", 25.00) + Item(3, "Mug", 25.00)
# should produce an Item with quantity 5, description "Mug", and price 25.00.

Design rule: Item.__add__() should know how to combine two item objects, but it should not search the cart.

TODO 2 — Maintain the No-Duplicate-Items Cart Invariant Invariant

Modify ShoppingCart.buyItem() so the cart never contains duplicate items with the same description and price.

Instead of always appending the new item, search the existing cart:

  • If the same item is already in the cart, combine the two items using __add__.
  • If the item is not already in the cart, append it normally.
# Cart invariant:
# The cart should never contain two Item objects
# with the same description and price.

Design rule: the cart is responsible for maintaining the no-duplicate-items invariant.

5. Reference Section

class ShoppingCart

Represents one shopping cart. A shopping cart has a name and a list of Item objects.

ShoppingCart name → string items → list of Item objects

__init__(self, name)

Creates a new empty cart. The cart begins with a name and an empty item list.

name property

Returns the cart name. The cart name is also used as the key in the shoppingcarts dictionary.

items property

Returns the list of items currently in the cart. This list is part of the cart's state because it changes when items are bought or when checkout occurs.

displayCart(self)

Displays the current contents of the cart. This method observes state but does not change it.

getItemNumber(self)

Gets a valid item number from the user. The user sees choices 1 through 10, but the inventory dictionary uses keys 1001 through 1010.

user input: 1 ↓ runtime inventory key: 1001

getQuantity(self)

Gets a positive integer quantity from the user. If the user enters x, the method returns None to signal that buying should be cancelled.

buyItem(self, inventory)

This is a transition method. It changes the cart state by adding an item or by increasing the quantity of an existing item. After TODO 4 is completed, it should preserve the no-duplicate-items invariant.

checkOut(self)

This is a transition method. It computes the total and changes the cart from a cart with items into an empty cart.

class Item

Represents one type of item in the cart.

Item quantity → integer description → string price → float

__init__(self, quantity, description, price)

Creates an item object with a quantity, description, and price.

__str__(self)

Returns a readable string representation of the item.

__add__(self, other)

Combines two compatible items and returns a new item. This supports the cart invariant that duplicate items should be merged rather than stored as separate objects.

File and Controller Functions

loadInventory()

Loads inventory from inventory.json. Because JSON keys are strings, this function converts inventory keys to integers before returning the inventory dictionary.

displayInventory(inventory)

Displays the store inventory to the user. It presents the inventory using menu numbers 1 through 10, even though the runtime inventory keys are 1001 through 1010.

loadShoppingCarts()

Loads shopping_carts.json and reconstructs runtime objects.

JSON dictionaries/lists ↓ ShoppingCart objects ↓ Item objects

storeShoppingCarts(shoppingcarts)

Converts runtime objects into JSON-compatible dictionaries and lists, then saves them.

ShoppingCart objects ↓ plain dictionaries/lists ↓ shopping_carts.json

cartName(shoppingcarts)

Lets the user load an existing cart or create a new cart. It returns one ShoppingCart object.

shoppingMenu()

Main controller. It loads data, lets the user choose a cart, displays the menu, calls cart methods, and saves carts after cart state changes.

6. Data Representation Summary

Runtime Representation

shoppingcarts dictionary key: cart name string value: ShoppingCart object name: string items: list of Item objects Item: quantity: int description: str price: float

Persistent JSON Representation

shopping_carts.json dictionary key: cart name string value: list of item dictionaries item dictionary: quantity description price

Important Distinction

Layer Representation Reason
JSON file Plain dictionaries and lists JSON can store simple structures, not Python objects.
Runtime program ShoppingCart and Item objects Objects organize behavior and state together.
Inventory JSON keys Strings such as "1001" JSON object keys are strings.
Runtime inventory keys Integers such as 1001 Integer keys make item-number calculations simpler.
