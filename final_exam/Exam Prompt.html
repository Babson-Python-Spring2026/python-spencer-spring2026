Final Exam: Required AI Prompt Instructions

You may use AI during the final exam, but the transcript must show that you are doing the computational thinking. AI may help you implement, debug, and understand code, but it should not do the design thinking for you.

For each exam problem, you must submit your Python code and your AI chat transcript.

Submission Requirements

Required Item What It Must Show
Python code for each problem Your code must run and must be manually commented by you. Do not ask AI to add the comments.
AI chat transcript for each problem The transcript must show that you understood the problem, made thoughtful design decisions, and identified inputs, state, transitions, invariants, and data representation.
Use of workbook material Your AI-generated code must stay within the Python concepts covered in the foundations workbook, except for the explicitly allowed OOP problem.
Important: Before asking AI for help, upload the Foundations of Programming in Python workbook to the AI chat. Then tell AI that all code must stay within the concepts covered in that workbook, except for the specific OOP concepts allowed in the shopping cart problem.

What a Strong Transcript Should Show

A strong transcript shows that you led the solution. AI may help implement the code, but you should be the one making the design decisions.

  • You explain the problem in your own words.
  • You identify the inputs.
  • You identify the state that changes over time.
  • You describe the transitions that change the state.
  • You identify invariants that must always remain true.
  • You explain your data representation choices.
  • You ask AI to check your reasoning, not replace your reasoning.
  • You can explain the final code in your own words.

Manual Code Comments

Your submitted code must be commented, but the comments must be written by you, not by AI.

The purpose of the comments is to show that you understand the code. A useful comment explains why a section of code exists, what state it changes, or how it protects an invariant.

Good comment style

# Update the running total after each item is processed.
# total is part of the state because it changes during the loop.
total = total + price

Weak comment style

# Add price to total
total = total + price

Bottom Line

AI may help you write code, but it should not be doing the thinking behind the solution. Your transcript and your manual comments should make your understanding visible.
